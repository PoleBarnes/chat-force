# Evaluation Criteria

The Mechanic uses these criteria to evaluate prototyping sessions.

## Must-pass (every deliverable)

- Output directly addresses the stated goal — no tangents
- No hallucinated facts, URLs, statistics, or quotes
- Code runs without errors (if code was produced)
- Follows existing project conventions (language, style, structure)
- Respects the never-list completely

## Quality signals (nice to have)

- Concise — no unnecessary verbosity or filler
- Reusable — could someone else pick up where this left off?
- Well-organized — clear structure, named files, logical layout
- Documented — explains non-obvious decisions (but only those)

## Orchestration quality

- Work was delegated to sub-agents rather than done directly
- Tasks were decomposed into focused sub-tasks
- Results were synthesized into a coherent deliverable
- Strategic decisions (approach, priorities) were made by the orchestrator
